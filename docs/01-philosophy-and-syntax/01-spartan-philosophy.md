# Спартанский подход: ключевые слова и builtin

Go был спроектирован в Google людьми, которые устали от сложности C++ (Кен Томпсон, Роб Пайк, Роберт Гризмер). Их цель — язык, который масштабируется не по выразительности, а по количеству инженеров и строк кода. Отсюда главный принцип: язык должен быть настолько маленьким, чтобы целиком помещаться в голове, а **код любого автора читался одинаково.**

В этом файле мы разберём, из чего состоит «спартанский» Go: его философию, полный список из 25 ключевых слов, встроенные функции и те синтаксические решения, которые сильнее всего удивляют человека, пришедшего из мира C#. Понимание этой философии важнее, чем знание синтаксиса: оно объясняет, почему в Go нет половины того, к чему вы привыкли — и почему его авторы считают это достоинством.

## Философия: минимализм как инженерное решение

C# исповедует подход «дайте разработчику много инструментов, пусть выбирает лучший». Go исповедует противоположный: «дайте один инструмент, чтобы выбирать было не из чего». Это не вопрос вкуса, а вопрос масштаба команды.

Ключевые принципы:

- **Ортогональность.** Возможности языка комбинируются предсказуемо и не пересекаются. Нет десяти способов объявить переменную или отфильтровать коллекцию. Меньше «фич» — меньше неожиданных взаимодействий между ними.
- **«Один способ сделать» (there's one way to do it).** Прямая противоположность перловому «there's more than one way». В C# отфильтровать список можно через LINQ-метод, через query-синтаксис, через цикл `foreach`, через `for`. В Go идиоматичный способ обойти коллекцию ровно один — `for ... range`.
- **Читаемость важнее краткости.** Код читают в разы чаще, чем пишут. Go сознательно жертвует лаконичностью записи ради того, чтобы любой инженер мог открыть незнакомый файл и сразу понять, что происходит, без знания хитрых идиом.
- **Скорость компиляции.** Go компилируется на порядки быстрее C++ и заметно быстрее крупных C#-решений. Быстрая сборка — это не удобство, а часть культуры разработки: цикл «правка → компиляция → тест» должен быть мгновенным. Ради этого язык отказался от заголовочных файлов, сложного вывода типов и тяжёлого метапрограммирования.
- **Единый форматтер (`gofmt`).** В мире C# существуют бесконечные споры о стиле: табы или пробелы, где ставить фигурную скобку, как выравнивать. Go закрыл эту тему навсегда: есть `gofmt`, он встроен в тулчейн, и его вывод — единственно правильный. Скобки только в стиле K&R (открывающая на той же строке), отступы — табы, и это не обсуждается. В .NET аналогом можно считать `.editorconfig` + `dotnet format`, но там это опционально и настраиваемо; в Go форматирование не настраивается вообще.

> **Параллель с .NET:** Если C# — это швейцарский нож с сотней лезвий (records, structs, classes, tuples, anonymous types, делегаты, события, LINQ, async-стримы), то Go — это один хорошо заточенный нож. Поначалу не хватает лезвий. Потом начинаешь ценить, что нож всегда открывается одинаково.

## 25 ключевых слов

В Go ровно **25 ключевых слов** — и это число зафиксировано в спецификации языка как часть его дизайна. Для сравнения: в C# их более 100 (около 80 зарезервированных плюс десятки контекстных вроде `async`, `await`, `var`, `record`, `nint`, `with`, `yield`). Вот полный список Go:

```
break        default      func         interface    select
case         defer        go           map          struct
chan         else         goto         package      switch
const        fallthrough  if           range        type
continue     for          import       return       var
```

Сгруппируем их по смыслу, чтобы было видно, как мало строительных блоков в языке:

- **Объявления:** `var`, `const`, `type`, `func`, `package`, `import`.
- **Управление потоком:** `if`, `else`, `for`, `switch`, `case`, `default`, `fallthrough`, `goto`, `break`, `continue`, `return`.
- **Конкурентность:** `go` (запуск горутины), `chan` (канал), `select` (мультиплексирование каналов), `defer` (отложенный вызов — тесно связан с управлением ресурсами и паникой).
- **Композиция типов:** `struct`, `interface`, `map`.
- **Итерация:** `range`.

Обратите внимание, чего здесь **нет** и быть не может, потому что в языке нет соответствующих концепций: `class`, `public`/`private`/`protected`, `static`, `async`/`await`, `try`/`catch`/`finally`/`throw`, `using`, `new` (как оператор-ключевое слово — в Go `new` это встроенная *функция*, см. ниже), `null`, `this`, `base`, `virtual`/`override`, `lock`, `event`, `delegate`, `namespace`. Каждое отсутствующее слово — это отсутствующая концепция, которую в Go либо решают иначе, либо не решают принципиально.

### Важное уточнение: что НЕ является ключевым словом

Это частый источник путаницы. Многие идентификаторы, которые выглядят как «зашитые в язык», на самом деле **предобъявленные идентификаторы** (predeclared identifiers), а не ключевые слова. Формально они объявлены во «вселенском» блоке (universe block) и их можно — теоретически, технически — переопределить (затенить) локальной переменной. Ключевое слово переопределить нельзя.

К предобъявленным идентификаторам относятся:

- **Константы:** `true`, `false`, `iota` (счётчик в блоках `const`).
- **«Нулевое значение» указателей и ссылочных типов:** `nil`. Да, `nil` — не ключевое слово, в отличие от `null` в C#.
- **Имена типов:** `int`, `int8`…`int64`, `uint`…`uint64`, `byte`, `rune`, `float32`, `float64`, `complex64`, `complex128`, `string`, `bool`, `error`, `any` (псевдоним `interface{}` с Go 1.18), `comparable`.
- **Встроенные функции:** `append`, `len`, `make` и остальные (см. следующий раздел).

```go
// Технически легально (компилируется!), но это антипаттерн ❌:
// мы затеняем предобъявленный идентификатор true локальной переменной.
func surprising() {
    true := false
    _ = true // здесь true == false
}

// А вот так нельзя — for это ключевое слово, не идентификатор:
// for := 5  // ошибка компиляции ❌
```

> **Параллель с .NET:** В C# `null`, `true`, `false` — это ключевые слова (литералы), их нельзя использовать как имена. В Go `nil`, `true`, `false` — это идентификаторы из стандартной библиотеки языка. Практической пользы в их переопределении нет, но это уточнение помогает понять, насколько действительно маленькое ядро у языка: даже базовые типы и `nil` не «вшиты» в грамматику, а «всего лишь» предобъявлены.

## Встроенные функции (builtin)

Раз в языке нет богатой стандартной библиотеки коллекций и нет дженерик-методов «из коробки» (как `List<T>.Add`), базовые операции над встроенными типами реализованы как **встроенные функции** — они доступны без импорта и обладают «магическими» свойствами (например, работают с разными типами, что обычным функциям до Go 1.18 было недоступно). Сгруппируем их по назначению.

**Работа со слайсами и мапами:**

- `make(T, ...)` — создаёт и инициализирует слайс, мапу или канал. Это единственный способ получить рабочую (не `nil`) мапу или канал.
- `new(T)` — выделяет память под `T`, обнуляет её и возвращает `*T` (указатель). В отличие от C#, `new` тут функция, а не оператор, и используется реже, чем кажется (обычно берут адрес через `&`).
- `append(s, elems...)` — добавляет элементы в слайс, при необходимости реаллоцируя backing-массив. Возвращает новый слайс (его обязательно надо присвоить). Подробно — в файле о слайсах.
- `copy(dst, src)` — копирует элементы между слайсами, возвращает число скопированных.
- `len(v)` — длина (строки, слайса, мапы, канала, массива).
- `cap(v)` — ёмкость (слайса, канала, массива).
- `delete(m, k)` — удаляет ключ из мапы.
- `clear(v)` *(Go 1.21+)* — очищает мапу (удаляет все ключи) или обнуляет все элементы слайса.

**Конкурентность:**

- `close(ch)` — закрывает канал (сигнал «данных больше не будет»).

**Обработка паник (аналог исключений для исключительных ситуаций):**

- `panic(v)` — инициирует панику (раскрутку стека). Используется для невосстановимых ошибок, **не** как обычный `throw`.
- `recover()` — перехватывает панику внутри `defer`-функции. Грубый аналог `catch`, но применяется крайне редко.

**Сравнение и упорядочивание (Go 1.21+):**

- `min(x, y, ...)` и `max(x, y, ...)` — минимум/максимум из произвольного числа упорядочиваемых аргументов. До 1.21 приходилось писать их вручную или тащить из библиотек.

**Комплексные числа (узкоспециальные):**

- `complex(r, i)` — собирает комплексное число, `real(c)` — действительная часть, `imag(c)` — мнимая.

**Отладочный вывод (использовать в реальном коде ❌):**

- `print(...)`, `println(...)` — примитивный вывод в stderr без форматирования. Для настоящего вывода используют пакет `fmt`. Эти builtin'ы существуют для нужд рантайма и отладки и могут быть удалены из языка — в коде их применять не следует.

```go
s := make([]int, 0, 8)   // слайс: len=0, cap=8
m := make(map[string]int) // рабочая мапа
s = append(s, 1, 2, 3)    // append возвращает новый слайс
fmt.Println(len(s), cap(s)) // 3 8
fmt.Println(min(3, 1, 2), max(3, 1, 2)) // 1 3 (Go 1.21+)
clear(m)                  // мапа снова пуста (Go 1.21+)
```

> **Параллель с .NET:** В C# `Add`, `Count`, `Remove`, `Min`, `Max` — это методы типов и LINQ-расширения. В Go их роль играют свободные встроенные функции, потому что встроенные типы (`[]T`, `map[K]V`) не имеют методов. Это следствие минимализма: вместо «всё есть объект с методами» — «есть данные и есть функции над ними».

## Базовый синтаксис: что удивит C#-разработчика

Даже не вдаваясь в семантику, человек из мира C# спотыкается о несколько синтаксических решений. Перечислим главные.

### Типы пишутся ПОСЛЕ имени

```go
var count int          // не "int count"
var name string = "Go" // тип можно указать явно
func add(a, b int) int { return a + b } // параметры, потом тип возврата
```

```csharp
int count;             // C#: сначала тип
string name = "Go";
int Add(int a, int b) => a + b;
```

Это не каприз: при чтении слева направо «имя, затем тип» лучше читается для сложных типов (например, указателей на функции) и упрощает грамматику. Если у нескольких параметров одинаковый тип, его указывают один раз: `func add(a, b int)`.

### Два способа объявить переменную: `var` и `:=`

```go
var x int = 10   // полная форма
var y = 10       // тип выводится (int)
z := 10          // короткая форма, только внутри функций
```

Короткое объявление `:=` — это `var` + вывод типа, доступное **только внутри функций**. На уровне пакета используется только `var`. Здесь Go нарушает собственный принцип «один способ сделать», но это исторический компромисс ради удобства.

> **Параллель с .NET:** `:=` похож на `var x = 10`, а полная форма `var x int` — на явный `int x`. Разница в том, что в Go вывод типа намеренно ограничен локальным контекстом и не претендует на «всеведение» C#-вского `var`.

### Нет тернарного оператора

В Go **нет** `?:`. Совсем. Это сознательное решение: авторы считают тернарники источником нечитаемого вложенного кода. Вместо него — обычный `if`:

```go
// Go: только полный if
max := a
if b > a {
    max = b
}
// либо встроенный max() с Go 1.21:
max := max(a, b)
```

```csharp
// C#: тернарный оператор
var max = b > a ? b : a;
```

### Единый `for` — нет `while`, `do-while`, `foreach`

В Go одно ключевое слово цикла — `for` — покрывает все случаи. Никаких `while` и `do`:

```go
for i := 0; i < 10; i++ { }      // классический for
for condition { }                 // аналог while
for { }                           // бесконечный цикл (аналог while(true))
for i, v := range slice { }       // аналог foreach с индексом
for k, v := range myMap { }       // обход мапы
for i := range 10 { }             // range по int (Go 1.22+): i = 0..9
```

Последняя форма — `for i := range 10` — появилась в Go 1.22 и заменяет привычное `for i := 0; i < 10; i++` для простого счётчика.

> **Параллель с .NET:** `for ... range` — это `foreach`, но он же закрывает и `for`, и `while`. Один цикл вместо четырёх конструкций — это и есть «один способ сделать».

### Множественный возврат и обработка ошибок

Функции в Go штатно возвращают несколько значений. Именно так реализована обработка ошибок — без исключений:

```go
func divide(a, b int) (int, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

result, err := divide(10, 2)
if err != nil {
    // обработка ошибки прямо здесь
}
```

```csharp
// C#: исключение либо out-параметр / Tuple
int Divide(int a, int b) {
    if (b == 0) throw new DivideByZeroException();
    return a / b;
}
```

Этой теме целиком посвящён Раздел 14, но синтаксически важно сразу привыкнуть: `value, err :=` — самый частый паттерн в Go-коде.

### `defer` — отложенное выполнение

`defer` откладывает вызов функции до момента выхода из текущей функции. Это идиоматичная замена `try/finally` и `using` для освобождения ресурсов:

```go
f, err := os.Open("file.txt")
if err != nil {
    return err
}
defer f.Close() // выполнится при выходе из функции, что бы ни случилось
// ... работа с f
```

```csharp
// C#: using или try/finally
using var f = File.OpenRead("file.txt");
// ... работа с f, Dispose вызовется автоматически
```

### Видимость через регистр первой буквы

В Go нет модификаторов `public`/`private`. Видимость идентификатора (для других пакетов) определяется регистром его первой буквы: с заглавной — экспортируется (публичный), со строчной — пакетно-приватный.

```go
func Exported()   {} // виден другим пакетам (как public)
func unexported() {} // виден только внутри своего пакета (как internal/private)
```

Это решение пронизывает весь язык: имена типов, полей структур, функций, методов, переменных — всё подчиняется одному правилу. Детально — в Разделе 4, здесь достаточно знать сам факт.

> **Параллель с .NET:** Заглавная буква ≈ `public`, строчная ≈ `internal` (видимость на уровне сборки/пакета). Прямого аналога `private` для члена типа нет — единица инкапсуляции в Go это пакет, а не тип.
